```

**Po wejściu na VM:**
```

```

**Okna w tmux** (zmień klawiszem `Ctrl-b <cyfra>`):
- `Ctrl-b 0` — bash
- `Ctrl-b 1` — keepalive

### Scenariusz B — VNC (graficzny pulpit XFCE)

Trzeba **DWA równoległe okna terminala na Macu**:

**Okno #1 — tunel SSH (zostaje otwarte cały czas):**
```

Nie wpisuj nic w tym oknie — to tylko tunel. Gdy je zamkniesz, VNC straci połączenie.

Po wejściu — XFCE desktop 1600×900, depth 24.

### Troubleshooting

**„Address already in use" przy próbie tunelu SSH:**
```
bind [127.0.0.1]:5901: Address already in use
```
To znaczy że poprzedni tunel SSH zombie nadal trzyma port 5901 na Macu.

**Na Macu**, w nowym oknie:
```
lsof -nP -iTCP:5901 -sTCP:LISTEN
```
Pokaże PID procesu trzymającego port. Zazwyczaj to stara `ssh`. Zabij:
```
kill <PID>
```
Potem powtórz scenariusz B.

**„Permission denied (publickey)" przy SSH:**
- Sprawdź czy klucz istnieje **na Macu**: `ls -la ~/.ssh/claude-vm_key.pem`
- Permisje muszą być 600: `chmod 600 ~/.ssh/claude-vm_key.pem`
- Komenda SSH musi być wpisywana **na Macu, nie na VM**. Klucz nie istnieje na VM!

**„Identity file ... No such file or directory":**
Znaczy że uruchamiasz `ssh` z VM zamiast z Maca. Wyjdź z VM (`exit`) i uruchom w nowym oknie terminala **na Macu**.

**Zapomniałeś hasła VNC:**
Na VM (przez SSH):
```
vncpasswd ~/.vnc/passwd
```
Wpisz nowe hasło 2×. Restart Xtigervnc:
```
pkill Xtigervnc
# automatycznie się wskrzesi przez systemd (sprawdź: systemctl --user status tmux-main.service)
# albo ręcznie:
vncserver -localhost -geometry 1600x900 -depth 24 :1
```
